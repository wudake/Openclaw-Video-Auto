# 3人团队使用方案

## 架构设计

```
┌─────────────┐     ┌─────────────┐     ┌─────────────────┐
│   用户A     │     │   用户B     │     │   用户C         │
│ (浏览器)    │     │ (浏览器)    │     │ (浏览器)        │
└──────┬──────┘     └──────┬──────┘     └────────┬────────┘
       │                   │                     │
       └───────────────────┼─────────────────────┘
                           │
                    ┌──────┴──────┐
                    │   Flask     │
                    │  (多线程)   │
                    └──────┬──────┘
                           │
              ┌────────────┼────────────┐
              │            │            │
       ┌──────┴──┐  ┌──────┴──┐  ┌──────┴──┐
       │ 用户A目录│  │ 用户B目录│  │ 用户C目录│
       │ videos/a/│  │ videos/b/│  │ videos/c/│
       │ output/a/│  │ output/b/│  │ output/c/│
       └─────────┘  └─────────┘  └─────────┘
```

## 实现方案

### 1. 用户认证（最简单方式）

**方案A: URL参数区分**（最快实现）
```
http://ip:5000/?user=a  → 用户A
http://ip:5000/?user=b  → 用户B
http://ip:5000/?user=c  → 用户C
```

**方案B: 简单登录页面**（推荐）
- 预设3个账号密码
- Flask Session 保持登录状态

### 2. 目录隔离

```
Dake-Video-Auto/
├── users/
│   ├── user_a/
│   │   ├── videos/raw/
│   │   ├── videos/configs/
│   │   └── output/
│   ├── user_b/
│   │   ├── videos/raw/
│   │   ├── videos/configs/
│   │   └── output/
│   └── user_c/
│       ├── videos/raw/
│       ├── videos/configs/
│       └── output/
```

### 3. 任务队列

使用 Python `queue.Queue` + `threading`：
- 每个用户任务进入队列
- 后台线程依次处理
- WebSocket 或轮询显示进度

### 4. 多线程支持

```python
# Flask 开启多线程
app.run(threaded=True)  # 改为 True
```

## 开发工作量

| 功能 | 工作量 | 说明 |
|------|--------|------|
| 用户登录 | 2小时 | 简单Session认证 |
| 目录隔离 | 3小时 | 所有路径改为动态 |
| 任务队列 | 4小时 | Queue + 后台线程 |
| 进度显示 | 2小时 | WebSocket或轮询 |
| **总计** | **11小时** | 约1.5天 |

## 技术方案

### 数据库（轻量级）
使用 SQLite，单文件存储：
- 用户表
- 任务表
- 配置表

### 部署
单台服务器即可：
- 4GB RAM
- 2核 CPU
- 20GB 磁盘

## 使用流程

```
1. 登录 → 选择用户A/B/C
2. 提交小红书链接 → 进入队列
3. 后台处理视频 → 实时显示进度
4. 完成后通知 → 下载视频
```

## 限制

- 同时只能处理1个视频（避免资源争用）
- 3人共用素材库（Logo/BGM）
- 无权限细分（所有人权限相同）
